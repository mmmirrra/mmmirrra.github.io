### Internet? Web?   
// 인터넷? 웹?

### Internet   
   
- Derived by ARPANET   
- <b>INTER</b>connected <b>NET</b>work → <b>INTERNET</b>   
- Worldwide Connected Computer Network System   
   
// TCP/IP (Transmission Control Protocal / Internet Protocol) 라는 전송규약을 사용하여 연결된 모든 네트워크   
// 1989년 웹 서비스의 개발로 인터넷의 사용 계기 마련됨   
// 초기의 인터넷은 키보드를 이용한 텍스트 정보 제공   
// 1993년 마우스와 모자이크 (Mosaic) 라는 웹 브라우저의 개발로 인터넷이 활성화됨   
- All networks connected using the TCP/IP (Transmission Control Protocal / Internet Protocol) protocol   
- The development of web services in 1989 provided an opportunity to use the Internet   
- Early Internet provides text information using keyboards   
- The Internet was activated in 1993 with the development of a web browser called Mosaic   
   
<br />
### Development of web   
// 웹의 발전   
   
// 인터넷 환경의 변화와 인터넷을 통한 다양한 멀티미디어 정보 사용의 활성화   
// - 인터넷상의 정보교환을 위한 최초의 분산형 멀티미디어 하이퍼텍스트 시스템 개발   
// -- 1989, 스위스 CERN 연구소의 '팀 버너스 리 (Tim Berners-Lee)' 등   
// - 개발 당시에는 사용자들의 플랫폼이 서로 달라 정보의 송수신이 원활하지 않음   
// 1993년 GUI 기반의 웹 브라우저 모자이크 (Mosaic) 개발   
// - 플랫폼에 상관없이 멀티미디어 정보를 수신 가능   
// - 이후 다양한 기능을 가진 브라우저들의 개발   
// - 모자이크 (Mosaic) : NCSA (National Center for Supercomputing Applications) 의 마크 안드레센 등에 의해 개발   
// 웹에 관련된 기술은 웹 컨소시엄 (W3C) 에서 개발   
// - '팀 버나스 리'를 중심으로 1994년 설립   
// - 웹의 상호 운용성을 목표로 함   
- Changes in the Internet environment and activation of the use of various multimedia information through the Internet   
  - Development of the First Distributed Multimedia Hypertext System for Information Exchange on the Internet   
    - Tim Berners-Lee and others at the CERN Institute in Switzerland, 1989   
  - At the time of development, users' platforms are different, so information transmission and reception are not smooth   
- Development of a GUI-based web browser Mosaic in 1993   
  - Receive multimedia information regardless of platform   
  - Later, the development of browsers with various functions   
  - Mosaic : Developed by Mark Andreessen and others of the NCSA (National Center for Supercomputing Applications)   
- Web-related technologies are developed by Web Consortium (W3C)   
  - Founded in 1994 with a focus on 'Tim Berners-Lee'   
  - Aim for Web interoperability   
   
<br />
### WWW (World Wide Web)   
   
- by Tim Berners-Lee   
   
// 하이퍼링크 개념과 GUI를 사용하여 다양한 형태의 데이터와 정보에 접근할 수 있는 인터넷 서비스   
// HTML로 작성된 웹 문서를 HTTP 프로토폴 기반으로 브라우저를 통해서 내려 받아서 읽을 수 있도록 구성됨   
- Internet services that use hyperlink concepts and GUIs to access different types of data and information   
- Configured to download and read HTML-written web documents through a browser based on HTTP Protopole

### Web Browser   
// 웹 브라우저   
   
// 인터넷망에서 정보를 검색하는 데 사용하는 응용 프로그램   
// 서버와 접속하여 HTTP 서버에 이용할 수 있도록 만든 하이퍼텍스트 문서를 읽고, 그것을 사용 중인 컴퓨터에 보여주는 프로그램   
// 웹 서비스에서 문자나 화상은 물론 동영상, 음악, 게임 등을 지원하는 멀티미디어 검색 프로그램   
// 구체적인 자료의 검색, 저장, 전송 등을 가능하게 함   
- An application used to search information from the Internet   
- A program that connects to a server and reads a hypertext document that is made available to the HTTP server and shows it to computer   
- Multimedia search programs that support text and video, video, music, games, etc. on web services   
- Enables search, storage, and transfer of specific data

### Web-related Terms   
// 웹 관련 용어   
   
// 웹 사이트 : 웹 페이지들의 의미 있는 묶음   
// 웹 문서 (HTML 문서) : 브라우저로 볼 수 있는 형식의 문서   
// 웹 페이지 : 각 브라우저 창에 표시되는 개별적인 문서   
// - 하나의 웹 문서는 주제/내용에 따라 여러 웹 페이지 파일로 구분/작성   
// 하이퍼텍스트 : 단위 텍스트들과 이를 결합하는 하이퍼링크로 구성된 텍스트   
// 웹 서버 : 웹 서비스를 제공하는 컴퓨터 시스템   
// 웹 클라이언트 : 웹 서비스를 제공받아 사용하는 사용자/시스템   
// - 웹은 서버 ↔ 클라이언트 구조에서 동작   
// HTTP : 웹 서버와 웹 클라이언트 간의 필요한 데이터를 주고 받기 위한 통신 규약   
// - 웹 브라우저에서 URL 입력 → DNS 서버는 URL을 IP 주소로 변환하여 웹 브라우저에게 전달 → 웹 브라우저는 웹 서버에게 HTML 페이지를 요청 → 웹 서버는 페이지를 요청 → 찾은 페이지를 웹 서버에게 전달 → 웹 서버는 HTML 파일을 웹 브라우저에게 전송 → 브라우저는 HTML 요소를 분석 및 변환하여 화면에 출력 → 사용자는 브라우저에서 HTML 파일을 볼 수 있음   
- Web site : A meaningful bundle of web pages   
- Web documents (HTML documents) : Documents in the form that can be viewed in a browser   
- Web page : Individual documents displayed in each browser window   
  - A single web document is created in multiple web page files depending on the topic and content   
- Hypertext : Text consisting of unit texts and hyperlinks that combine them   
- Web Server : Computer systems that provide web services   
- Web Client : Users/Systems that receive and use web services   
  - Web operates on Server ↔ Client structure   
- HTTP : Conventions for communication between web servers and web clients to send and receive necessary data   
  - Enter a URL in a web browser   
    → A DNS server converts a URL to an IP address and forwards it to a web browser   
    → A web browser requests an HTML page from a web server   
    → A web server requests a page   
    → Forwards a found page to a web server   
    → A web server sends an HTML file to a web browser   
    → The browser analyzes and converts HTML elements and outputs them to the screen   
    → Users can view the HTML file in the browser

### Web 2.0   
// 웹 2.0   
   
// 웹 1.0 (기존의 웹) : 일반적인 정보 전달   
// 웹 2.0 : 새로운 사용자 중심의 인터넷 환경   
- Web 1.0 (Traditional Web) : General Information Delivery   
- Web 2.0 : New User-Centered Internet Environment   
   
// 사용자들이 각종 콘텐츠를 자유롭게 올리고, 인터넷 서비스를 직접 만들 수 있게 하는 등 이용자의 적극적인 참여를 유도하고 정보 공유를 확대하려는 것   
// 2004년 팀 오라일리 (Tim O'Reilly) 에 의해 사용된 용어   
- It is intended to encourage active participation of users and expand information sharing by allowing users to freely upload various contents and create Internet services themselves   
- 2004 term used by 'Tim O'Reilly'   
   
// 웹 2.0 의 특징   
// - 플랫폼으로서의 웹   
// - 개방과 참여   
// - 집단 지성의 활용   
// - 가벼운 프로그래밍 모델   
// - 풍부한 사용자 인터페이스   
- Features of Web 2.0   
  - Web as a platform   
  - Openness and engagement   
  - The use of collective intelligence   
  - A light programming model   
  - Many user interface   
   
// 웹 2.0 기술과 활용   
// - 콘텐츠 신디케이션 (contents syndication)   
// -- 콘텐츠 공급자와 수요자 사이를 연결해 주는 콘텐츠 중계 사업   
// -- 중계사 (신디케이터) 는 콘텐츠를 직접 생산하지는 않지만 우수한 콘텐츠를 수집, 가공해 인터넷 운영업체에 제공   
// -- 표준화된 프로토콜을 이용   
// --- 최종 사용자가 사이트의 자료를 다른 장소 즉, 다른 웹 사이트, 독립된 PC 등에서 이용할 수 있도록 함   
// -- 신디케이션을 위한 프로토콜은 모두 XML 을 기반으로 함   
// -- 대표적인 프로토콜은 RSS, Atom 등   
- Web 2.0 technology and utilization   
  - Contents syndication   
    - A content relay business that connects content providers and consumers   
    - The relay (syndicator) does not produce content directly, but collects and processes excellent content and provides it to Internet operators   
    - Using standardized protocols   
      - Enables end users to use the site's material in another location, such as another website, an independent PC, etc   
    - Protocols for syndication are all XML based   
    - Representative protocols include RSS, Atom, etc   
   
// RSS (Really Simple Syndication, Rich Site Summary)   
// - 자주 업데이트 되는 웹 사이트 간의 콘텐츠를 교환하기 위한 XML 기반의 표현 형식   
// - 뉴스나 블로그에서 주로 사용하는 콘텐츠 표현 방식   
// - 해당 사이트에 접속하지 않고도 RSS 피드를 통해 정보가 갱신   
// - 팟캐스팅 (PodCasting) 과 같은 미디어 배포 용도로 사용   
- RSS (Really Simple Syndication, Rich Site Summary)   
  - XML-based representation format for exchanging content between frequently updated websites   
  - The way content is expressed in news or blogs   
  - RSS feeds update information without access to the site   
  - Use for media distribution, such as Podcasting   
   
// 소셜 태깅 (Social Tagging)   
// - 사용자가 키워드를 임의의 태그 (tag) 로 붙이고 이들을 검색하는 기법   
// -- 기존의 웹 검색의 경우는 미리 정해져 있는 카테고리에 따라 자료를 분류하고 검색하는 디렉토리 방식의 검색 방법   
// - 사용자가 자유롭게 붙이는 태그는 그대로 레이블링을 되어 정보가 분류   
// - 태그는 자료에 원하는 수만큼 붙일 수 있기 때문에 태그를 이용한 검색은 다양한 내용의 검색이 가능   
// - 태그를 이용한 대표적인 서비스   
// -- 소셜 북마킹 사이트 : 딜리셔스 (del.icio.us)   
// -- 온라인 사진 공유 커뮤니티 사이트 : 플리커 (Flickr)   
- Social Tagging   
  - A technique in which users attach keywords to tags and search for them   
    - In the case of existing web search, a directory method of classifying and searching data according to a predetermined category is used   
  - Tags that users attach freely are labeled as they are, and information is classified   
  - Tags can be attached to data as many times as you want, so searching using tags can search for various contents   
  - Representative services using tags   
    - Social Bookmarking Site : Delicious (del.icio.us)   
    - Online Photo Sharing Community Site : Flickr   
   
// 공개 API와 매쉬업   
// - API (Application Program Interface)   
// -- 다른 응용프로그램을 개발할 때 해당 모듈을 쉽게 사용할 수 있도록 해주는 개발 환경   
// - 공개 API   
// -- 자신의 사이트에서 제공되는 웹 서비스 기능을 다른 사이트에서 활용할 수 있도록 API를 제공해 주는 것   
// -- 구글 맵스 (Google Maps) 나 아마존 (Amazon) 의 상품 검색 및 구매 기능 등의 공개 API 제공   
// - 매쉬업 (Mashup)   
// -- 공개 API를 이용하여 두 가지 이상의 웹 서비스를 조합하여 새로운 웹 서비스를 제공하는 방법   
- Public API and Mashup   
  - API (Application Program Interface)   
    - A development environment that makes the module easier to use when developing other applications   
  - Public API   
    - Providing APIs so that web service functions provided by their sites can be utilized by other sites   
    - Providing public APIs such as product search and purchase functions on Google Maps or Amazon   
  - Mashup   
    - How to provide a new web service by combining two or more web services using a public API   
   
// 소셜 네트워크의 FOAF 와 XFN   
// - 소셜 네트워크의 목적으로 사용되는 특수 프로토콜   
// - 사이트의 기능을 확장시켜 주거나 중앙에서 관리하는 웹 사이트 없이도 최종 사용자들의 상호작용을 가능케 함   
// -- FOAF (Friend of a Firend)   
// --- 시맨틱 웹 (Semantic Web) 기술을 적용하여 관계성을 확장하는 기술   
// --- 몇 단계를 거치면서 전 세계의 모든 사람을 알 수 있다는 다단계 구조를 이용   
// -- XFN (XHTML Friends Network)   
// --- 하이퍼링크를 이용하여 인간관계를 표현하는 수단   
// --- 블로그에서 다른 사람과의 관계를 표현할 수 있는 방법을 제공   
- FOAF and XFN on social networks   
  - Special protocols used for social network purposes   
  - Expanding the features of the site, or enables end-user interaction without the need for a centrally managed website   
    - FOAF (Friend of a Firend)   
      - Technology to extend relationships by applying Semantic Web technology   
      - Using a multi-level structure that allows you to know everyone in the world through several steps   
    - XFN (XHTML Friends Network)   
      - A means of expressing human relationships using hyperlinks   
      - Provides a way to express your relationship with others in your blog   
   
// 검색 서비스의 진화   
// - 검색 서비스 : 통신망에서 제공하는 정보검색을 위한 서비스   
// - 검색 엔진 : 정보 검색을 위한 소프트웨어 시스템   
// -- 여러 검색엔진을 통해 이루어지는 검색 서비스를 구글 (Google) 이 페이지랭크 개념을 도입함으로써 획기적으로 발전   
// --- 페이지랭크 (page rank)   
// ---- 스탠포드 대학의 래리 페이지와 세르게이 브린이 개발한 검색 기술   
// ---- 검색 결과를 보여줄 때 문서의 중요도에 따라 결과 페이지의 상위에 나열하는 개념   
// ---- 웹과 같이 하이퍼링크 구조를 가지는 문서에 상대적 중요도에 따라 가중치를 부여하는 방법   
// -- 웹 문서 검색 서비스 뿐만 아니라 구글 이미지 검색, 구글 맵, 구글 어스 등 다양한 검색 서비스로 발전   
- Evolution of search services   
  - Search service : A service for searching information provided by a communication network   
  - Search engine : Software system for information retrieval   
    - Google has dramatically advanced search services through multiple search engines by introducing the concept of page rank   
      - Page rank   
        - Search technology developed by Larry Page and Sergey Brin at Stanford University   
        - The concept of listing results at the top of the results page based on the importance of the document when displaying search results   
        - How to weight documents with hyperlink structures, such as the web, according to their relative importance   
    - In addition to web document search services, it develops into various search services such as Google Image Search, Google Maps, and Google Earth   
   
// 팻캐스팅 (PodCasting)   
// - 오디오 파일 또는 비디오 파일 형태로 뉴스나 드라마 등 다양한 콘텐츠를 인터넷 망을 통해 제공하는 서비스   
// - 애플의 아이팟 (iPod) 과 방송 (broadcasting) 을 합성한 신조어   
// -- 기존의 라디오 프로그램과 달리 방송시간에 맞춰 등을 필요가 없으며, MP3 플레이어나 스마트 폰 등을 통해 구독 등록을 하면 자동으로 업데이트 됨   
// - 팟캐스팅의 구독모델은 RSS 2.0 XML 이나 RDF XML 파일 형식을 사용하여 파일을 제공   
// -- RDF (Resource Description Framework)   
// --- 웹에 있는 자원에 관한 메타 정보를 표현하기 위한 언어   
// - 독립 제작자들은 자신만의 라디오 프로그램을 만들고 제공할 수 있게 되었으며, 최근에는 동영상 파일을 전달하는 팟캐스트가 다수 등장   
- PodCasting   
  - A service that provides various contents, such as news or dramas, through the Internet in the form of audio files or video files   
  - A new word that combines Apple's iPod and broadcasting   
    - Unlike conventional radio programs, there is no need to turn on your back in time for broadcasting, and it is automatically updated when you register a subscription through an MP3 player or smartphone   
  - Podcasting's subscription model uses RSS 2.0 XML or RDF XML file formats to provide files   
    - RDF (Resource Description Framework)   
      - Language for expressing meta information about resources on the web   
  - Independent producers are now able to create and deliver their own radio programs, and more recently, a number of podcasts have appeared delivering video files   
   
// UCC 또는 UGC   
// - UCC (User Created Contents)   
// -- 사용자가 직접 만든 동영상, 글, 사진 등의 제작물   
// -- 사용자가 상업적인 의도 없이 제작한 콘텐츠   
// -- UGC (User Generated Contents) 라고도 함   
- UCC or UGC   
  - UCC (User Created Contents)   
    - Production of videos, writings, photos, etc. created by users themselves   
    - Content created by users without commercial intentions   
    - Also known as UGC (User Generated Contents)   
   
// 위키피디아 (Wikipedia)   
// - 모두가 함께 만들어가며, 누구나 자유롭게 쓸 수 있는 다언어 인터넷판 백과사전   
// - 집단 지성으로 작성되는 대표적인 UCC   
- Wikipedia   
  - A multilingual Internet encyclopedia that everyone makes together and can write freely   
  - A representative UCC written with collective intelligence

- Key technologies in Web 3.0   
  - Semantic Web   
    - It means "semantic web"   
    - Semantic Web is a framework and technology that expresses the relationship (meaning information) between resources and information about resources such as web documents, various files, and services in a computer-processed ontology form in a distributed environment such as the current Internet, and allows computers to process them   
      - Ontology   
        - Representation of existing object and their relationships and various concepts in a form that computers can process   
        - Use in artificial intelligence, semantic web, natural language processing, etc   
        - Representative languages used for ontology writing : RDF, OWL, etc

### HTML Overview   
// HTML의 개요   
   
// 인터넷의 발전 : 웹의 역할이 결정적인 요인   
// - 웹의 핵심적인 요소 → HTML의 개발

// HTML (Hyper Text Markup Language)   
// - 웹 페이지를 작성할 수 있는 마크업 언어   
// - 웹 문서의 제목, 단락, 목록 등과 같은 본문을 위한 구조적 의미를 나타내고, 링크, 인용과 그 밖의 항목으로 구조적 문서를 만들 수 있는 방법을 제공   
// - '팀 버너스 리 (Tim Berners-Lee)' 에 의해 처음 제안   
// - 1992년 '버전 1.0' 을 시작으로 1999년 W3C 의 권고안 'HTML 4.01' 이 배포되고, 2008년 'HTML5' 의 초안이 배포되었음   
// - HTML은 SGML (Standard Generalized Markup Language) 에 기초   
// - 하이퍼텍스트 언어이면서 일반적인 문서의 특성을 가짐   
// - HTML 이 워드 문서와 다른 점   
// -- 태그 `< >` 를 사용하여 작성함   
// -- [Spacebar], [Enter], [Tab] 키를 인식하지 못함   
// -- HTML 문서는 반드시 텍스트 문서로 저장되어야 하며, 저장할 때는 `*.htm` 이나 `*.html` 의 확장자를 사용해야 함   
// -- 대, 소문자를 구분하지 않음   
- HTML (Hyper Text Markup Language)   
  - Markup Language for Creating Web Pages   
  - Indicates structural meanings for the body, such as the title, paragraph, list, etc. of a web document, and provides a way to create a structural document with links, citations, and other items   
  - First proposed by Tim Berners-Lee   
  - Starting with 'version 1.0' in 1992, 'HTML 4.01' of the W3C was distributed in 1999, and the draft of 'HTML5' was distributed in 2008   
  - HTML is based on SGML (Standard Generalized Markup Language)   
  - Hypertext language with common document characteristics   
  - How HTML Differences Word Document   
    - Created using tag `< >`   
    - [Spacebar], [Enter], [Tab] key not recognized   
    - HTML documents must be stored as text documents and must be stored with an extension of `*.htm` or `*.html` when stored   
    - Upper case, lower case not separated   
   
// SGML (Standard Generalized Markup Language)   
// - 1986년 ISO에 의해 기술적 마크업 언어로서 국제 표준으로 제정됨   
// - 모든 문서를 헤더 파일, 문서 형태 정의 (DTD), 본문 등 세 부분으로 구성   
// -- 절차적 마크업 언어   
// --- 메시지에 나타나는 텍스트의 포맷과 관련 사항을 기술하기 위한 언어   
// --- 특정한 편집 시스템이나 응용 환경에 제한적인 단점   
// -- 기술적 마크업 언어   
// --- 포맷되는 문헌 요소의 내용이 어떤 것인지를 기술하는 언어   
// --- 문헌의 처리 과정을 기술하는 것이 아니라 문헌의 논리적 구조와 속성을 기술   
// --- 따라서 특정한 편집 시스템이나 응용 환경에 제한을 받지 않음   
// - 문서의 구조를 정의할 수 있는 메타 언어로서의 국제 표준이지만 기능이 복잡하여 널리 쓰이지 못함   
// -- HTML 과 XML 의 근거를 제공   
- SGML (Standard Generalized Markup Language)   
  - Enacted as an international standard by ISO in 1986 as a technical markup language   
  - All documents consist of three parts : Header file, Document form definition (DTD), and Body   
    - Procedural markup language   
      - Language to describe the format and related matters of the text that appears in the message   
      - Disadvantages limited to certain editing systems or applications   
    - Technical markup language   
      - A language that describes what the contents of a formatted literature element are   
      - Not describing the processing of the literature, but describing the logical structure and properties of the literature   
      - Unlimited to specific editing systems or application environments   
  - International standard as a meta-language that can define the structure of a document. However, it is not widely used due to its complexity of features   
    - Provides a basis for HTML and XML   
   
// XML (eXtensible Markup Language)   
// - 확장 가능한 마크업 언어   
// - 구조화된 문헌을 웹 상에서 전송할 수 있도록 텍스트 형식으로 설계   
// -- 구조화된 HTML의 한계를 극복하고, SGML 의 복잡함을 해결하는 방안   
// -- HTML 에 사용자가 새로운 태그를 정의할 수 있는 기능이 추가된 표준   
// - XML 의 장점   
// -- 장치와 시스템에 독립적임   
// --- 다양한 소프트웨어와 하드웨어에서 이용될 수 있음   
// -- 매우 명확함   
// --- 선택기법의 수가 적고, 태그의 생략을 유발하는 태그 최소화가 없음   
// -- 이용자 공유의 태그를 만들 수 있음   
// -- 다양한 방법으로 문헌을 디스플레이 할 수 있음   
// -- 특정 언어에 독립적임   
// --- 언어나 국가에 제한 없이 사용될 수 있음   
- XML (eXtensible Markup Language)   
  - Extensible markup language   
  - Designed in text format to allow structured literature to be transferred over the web   
    - Overcoming the limitations of structured HTML and addressing the complexity of SGML   
    - A standard with HTML that allows users to define new tags   
  - Strengths of XML   
    - Independent of devices and systems   
      - Available in a variety of software and hardware   
    - Very clear   
      - Fewer selection techniques, No tag minimization that causes the tag to be omitted   
    - XML can create tags for user shares   
    - XML can display literature in a variety of ways   
    - Independent of a particular language   
      - Can be used without restriction to language or country   
   
// DHTML (Dynamic HTML)   
// - 기존의 HTML이 가진 정적인 문서를 동적이고 생동감있는 웹 페이지로 만드는 기술   
// -- 웹 문서의 모든 구성요소를 객체화한 다음 스크립트 언어를 통해 동적으로 보여주는 기술   
// - 스크립트 언어   
// -- 어떤 객체에 대한 특성이나 행동에 대해서 기술해 놓은 언어   
// -- 간단하고 사용이 편리한 반면, 독립적이고 복잡한 응용 프로그램을 만들기에는 부적합   
// -- 웹 페이지와 같은 간단한 텍스트로 이루어진 내용은 스크립트 언어를 사용하면 다양한 효과를 얻을 수 있음   
// - 웹 페이지에 사용되는 스크립트 언어 종류   
// -- 자바 스크립트   
// -- VB 스크립트   
// -- J 스크립트   
- DHTML (Dynamic HTML)   
  - Technology to make static documents from traditional HTML into dynamic, lively web pages   
    - Technology that objectifies all components of a Web document and then dynamically shows them through the script language   
  - Script language   
    - A language that describes a characteristic or behavior of an object   
    - Simple and easy to use, unsuitable for creating independent and complex applications   
    - Simple textual content, such as web pages, can have a variety of effects using script language   
  - The type of script language used for web pages   
    - JavaScript   
    - VB Script   
    - J Script

// - 가상현실 모델링 언어   
// - 사용자와 상호작용이 가능한 3차원 이미지 장면을 묘사하기 위한 언어

// - VRML 파일을 보기 위해서는 VRML 뷰어나 브라우저가 필요   
- JavaScript   
  - Developed by Netscape Communication   
    - Creating a Script Language to Run in a Web Browser   
  - Enables users to provide the information they want with dynamic capabilities   
    - HTML consists of tags that unilaterally provide information to users   
  - It can be inserted into HTML documents to create interactive web documents   
- DHTML   
  - A technique that combines HTML, JavaScript, and CSS, a style-defining language, to create an interactive web page   
  - CSS (Cascading Style Sheet)   
    - Standard for representing style sheet on web pages   
    - Style Sheet   
      - A collection of style rules that set the style of a document   
      - Separate the style and logical structure of a document, giving components of a web page a variety of styles, such as location or color   
- CGI (Common Gateway Interface)   
  - A method or protocol for exchanging information between a web server and an external program   
  - Use when a browser creates an interactive web page, such as sending a query to a database server via the server   
  - Standard way to run external programs   
    - Strengths : Easy to use   
    - Weaknesses : The server runs many programs, which puts pressure on the server   
- ASP (Active Server Pages)   
- PHP (Hypertext Preprocessor)   
- VRML (Virtual Reality Modeling Language)   
  - Virtual reality modeling language   
  - Language to describe a 3D image scene that can interact with the user   
  - Allows users to create visual images to interact with three-dimensional scenes such as viewing, moving, turning, etc., and put them into web pages   
  - A VRML viewer or browser is required to view a VRML file

  - JavaScript : Responsible for the behavior and interaction of web pages   
    - Dynamic handling of events and elements, etc.
